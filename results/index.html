Computation of a Germanium molecule using ksdft++

(from setup to results)

The model

As a walkthrough example we calculate the electronic configuration of 8 Germanium atoms, with 32 valence electrons, consisting of 16 electrons in 2 spin configurations.

Configuration of the ionic potential

The ionic potential is hard coded in the ionic_potential class. To add a new atomic type one has to adapt the code and add a new potential expression. The following code shows the implementation of the model for a germanium core potential. The potential is called Starkloff Joannopoulos model

$$ V(r) = - { Z \over r } { 1 - e^{-\lambda r} \over 1 + e^{-\lambda(r-r_{c}) } } $$

T. Arias derived a Fourier series form which can be cut off after few terms and gives:

$$ V(G) = - { 4 \pi Z \over G^2 } + { 4 \pi Z (1+e^{-\lambda r_{c}}) \over G^2 } V_{2}(G) $$ $$ V_{2}(G) = \Bigg( -{2 \pi e^{-\pi G \lambda} cos(G r_{c}) {G \over \lambda} \over 1 - e^{-2 \pi G / \lambda} } + \Sigma_{n=0}^{\infty} (-1)^n { e^{-\lambda r_{c} n} \over 1 + ({n \lambda \over G} )^2 } \Bigg) $$

The ionic potential is implemented in the ionicPotentialClass class, which holds the geometrical quantities to compute an arbitrary ionic potential. Here the potential is defined as a quantity in wavenumber space and finally it is backtransformed to real space before it is stored in the class matrix Vdual.

The code checks the computed potential Vdual for flaws before it is turned over to the routine forming the energy functional expression.

Configuration of the input files

For every case we compute there has to be a folder in the case directory. For the Germanium case we find a folder case/Germanium where two files reside.

There are two files to adapt for ksdft++ to run:

  • the atoms.param file holds the positions of the atomic cores
  • the inputFile.param file holds physical, geometrical and other parameters

In every line the position of one ion core is inserted, giving 8 cores in this case. The units are bohr, which is around \( 5.3 \times 10^{-11} m \) The Germanium molecule is arranged in a diamond shaped grid.

Run the case

To run the case, given that the setup.sh configuration script has succeeded in building the executable, we can simply type:

./release_ksdft++ nameofcase

In this case this reads:

./release_ksdft++ Germanium

Postprocessing

As soon as the code has finished computing you'll find some files with ending 2dMat, which hold the 2D matrix slices to be plotted. In the latex.tex file there is the latex file assembled by the code, describing the computations which took place. In the file "finalResult_Germanium.dat" the energy leves of the various electrons is summarized.

To get to the finished latex pdf document and plot all the figures, execute the plotScript.sh file in this folder. It uses gnuplot to derive all the images and creates a folder structure in ./POSTPROCESSING/ where all the cases have a distinct folder with their names. In this folder, for Germanium it is ./POSTPROCESSING/Germanium/ we find a DATA folder, where the 2dMat files are stored. In a doc folder the latex pdf file is stored, named in this case as "Germanium_doku.pdf". In the IMG folder all the plots can be found.
